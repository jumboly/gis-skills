# 空間インデックスシステム リファレンス

## システム比較表

| システム | セル形状 | 階層 | 精度レベル数 | 外部依存 | 主な用途 |
|---------|---------|------|------------|---------|---------|
| Geohash | 矩形 | あり (prefix) | 1-12 | なし | DB空間インデックス (Elasticsearch/DynamoDB/Redis) |
| H3 | 六角形 | あり (parent/child) | 0-15 | h3 | データエンジニアリング (BigQuery/Snowflake/Databricks) |
| Plus Code (OLC) | 矩形 | なし | 2-15 | openlocationcode | 人間可読な位置コード (Google Maps) |
| Quadkey | 矩形 | あり (prefix) | 1-23 | なし | Bing Maps / CARTO Quadbin。XYZタイルと1:1対応 |
| MGRS | 矩形 | 限定的 | 0-5 | mgrs | NATO軍事標準。防災・捜索救助 |
| Maidenhead | 矩形 | あり (prefix) | 1-5 | なし | アマチュア無線グリッドロケーター |
| Morton code | − | なし | ビット深度指定 | なし | 空間充填曲線。Geohash/Quadkeyの基盤技術 |

## システム選択ガイド

### ユースケース別推奨

| ユースケース | 推奨システム | 理由 |
|------------|------------|------|
| Elasticsearch/DynamoDB の空間インデックス | **Geohash** | prefix 検索でレンジクエリ可能 |
| BigQuery/Snowflake での空間集約 | **H3** | 六角形グリッドで均一な近傍性、UDF対応 |
| ユーザーに位置を共有 | **Plus Code** | 人間可読、Google Maps対応 |
| Web地図タイルキャッシュ | **Quadkey** | XYZタイルと1:1、prefix=親タイル |
| 防災・捜索救助の現場座標 | **MGRS** | NATO標準、地形図で使用 |
| アマチュア無線の通信レポート | **Maidenhead** | 無線業界の標準グリッド |
| データベース内部の空間ソート | **Morton code** | 空間近接性を保つ1次元ソートキー |

### 選択フローチャート

```
六角形グリッドが必要？ → Yes → H3
  ↓ No
人間が読み書きする？ → Yes → Plus Code
  ↓ No
軍事/防災用途？ → Yes → MGRS
  ↓ No
Webタイルと連動？ → Yes → Quadkey
  ↓ No
アマチュア無線？ → Yes → Maidenhead
  ↓ No
DB空間インデックス/汎用？ → Yes → Geohash
  ↓ No
空間ソートキーが必要？ → Yes → Morton code
```

### 長所・短所比較

| システム | 長所 | 短所 |
|---------|------|------|
| Geohash | prefix検索可、実装容易、広く普及 | 矩形セルの辺境界問題、高緯度で歪み |
| H3 | 均一面積、良好な近傍性、六角形 | ライブラリ依存、コードが長い |
| Plus Code | 人間可読、Googleマップ統合 | 階層操作なし、近傍検索不可 |
| Quadkey | タイルと1:1対応、prefix=ズーム | Web Mercator限定、極地なし |
| MGRS | 高精度、国際標準 | 複雑な構造、UTMゾーン境界問題 |
| Maidenhead | シンプル、広く普及(無線) | 精度限定、一般利用は少ない |
| Morton code | 高速ビット演算、概念的にシンプル | 直接利用は少ない、人間非可読 |
